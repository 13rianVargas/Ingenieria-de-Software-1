# 4. Artefactos

## Personal Software Process (PSP)

#### Registro de Tiempos (Time Log)
Tracking por fase: planeación, diseño, revisión, codificación, pruebas. Base histórica para estimaciones futuras.

#### Registro de Defectos (Defect Log)
Captura defectos: tipo, fase introducida, fase detectada, esfuerzo. Insumo para calcular Yield (efectividad).

#### Plan de Proyecto Individual
Estimaciones de tiempo, esfuerzo, tamaño. Línea base personal comparada contra realidad.

#### Especificación de Requisitos
Clarifación escrita de qué se debe construir: objetivos, funcionalidades, restricciones.

---

## Team Software Process (TSP)

#### Plan de Lanzamiento (Launch Plan)
Definido colectivamente: objetivos, metas, cronograma, roles TSP, riesgos. Genera compromiso de equipo y línea base.

#### Plan de Ciclo (Cycle Plan)
Por iteración: tareas asignadas, estimaciones, dependencias. Validado en standup diario.

#### Especificación de Sistema (System Specification)
Requisitos y diseño acordados por equipo. Insumo único para todos los PSP.

#### Plan de Pruebas del Sistema
Casos de prueba de integración, matriz de trazabilidad, responsabilidades. Valida componentes juntos.

#### Reporte de Estado (Status Report)
Progreso semanal: tareas, defectos, riesgos, acciones correctivas. Métrica objetiva para decisiones.

#### Registros de Inspección Formal
Actas de revisión: defectos por tipo/severidad, tiempo, checklist. Base para mejora continua.
