Process Script — PSP0 (flujo del proceso)

Propósito: Define la secuencia de pasos que el desarrollador sigue en cada programa. Haz clic en cualquier paso para editarlo en tiempo real.

Fase 1 · Planning
Obtener los requerimientos del programa. Estimar el tamaño (LOC) usando datos históricos personales. Calcular el tiempo requerido con la tasa de productividad propia (LOC/hr). Registrar el plan en el Project Plan Summary. Si no hay datos históricos, usar estimación analógica.
Fase 2 · Design
Producir el diseño lógico completo del programa antes de escribir código. Documentar la solución usando pseudocódigo, diagramas de flujo o lógica estructurada. Registrar el tiempo utilizado en el Time Log. No iniciar la codificación hasta tener el diseño finalizado.
Fase 3 · Code
Implementar el diseño siguiendo los estándares de codificación definidos. Registrar todos los defectos encontrados durante la codificación en el Defect Log, incluyendo tipo, fase de inyección y tiempo de corrección. Registrar el tiempo en el Time Log.
Fase 4 · Compile
Compilar el programa. Corregir todos los errores de compilación. Registrar cada defecto encontrado y removido en el Defect Log con su tipo (tipos 10–100 de Humphrey). No continuar a la fase de pruebas hasta lograr compilación limpia.
Fase 5 · Test
Ejecutar las pruebas definidas en el plan de pruebas. Corregir todos los defectos encontrados registrándolos en el Defect Log. Repetir hasta pasar todas las pruebas exitosamente. Registrar tiempo en el Time Log. Documentar los casos de prueba utilizados.
Fase 6 · Postmortem
Completar el Project Plan Summary con los datos reales del programa. Calcular métricas de calidad (yield, defectos/KLOC) y productividad (LOC/hr). Identificar áreas de mejora. Almacenar todos los datos en la base de datos personal del proceso para calibrar estimaciones futuras.

* Haz clic en cualquier paso para editar su contenido durante la presentación.
