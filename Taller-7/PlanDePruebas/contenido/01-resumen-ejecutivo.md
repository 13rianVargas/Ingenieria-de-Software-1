# Resumen Ejecutivo

Este documento constituye el Plan de Pruebas de Software para el sistema
**E-Commerce Comercial Konrad**, desarrollado como entregable del Taller-7
de la materia Ingeniería de Software I.

El plan cubre dos historias de usuario del ciclo de onboarding del
marketplace:

- **HU-1 — Registro de Vendedor**: proceso mediante el cual una persona
  natural o jurídica solicita ser vendedor en la plataforma, adjuntando
  documentos y pasando por validación crediticia y judicial.
- **HU-2 — Publicar Productos**: proceso mediante el cual un vendedor
  aprobado y con suscripción activa registra un producto con sus atributos
  en la plataforma.

## Propósito del plan

Definir el alcance, enfoque, criterios y recursos necesarios para verificar
que las dos funcionalidades cubiertas se comportan según la especificación
del cliente (ver el enunciado del taller).

## Tipo de plan

Plan de pruebas detallado, de alcance funcional, asociado a dos HU del
Release 1 (Onboarding y Vitrina) del proyecto.

## Alcance del esfuerzo

- Número de casos de prueba diseñados: por definir.
- Criterio de completitud: 100 % de los casos de prueba ejecutados.
- Criterio de éxito mínimo: 0 casos con severidad critical o major abiertos.
